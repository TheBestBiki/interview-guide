# Q31: 介绍一个你最有挑战的项目？

## 项目背景

**嘉立创EFS（Electronic Finance System）应收账款结算中台**

- **业务背景**：为嘉立创供应链上下游提供电子保理、应收账款转让、融资撮合等服务
- **系统规模**：日均交易额5000万+，用户数10万+，QPS峰值2万
- **技术挑战**：高并发、资金安全、多系统集成、数据一致性

## 核心挑战

### 1. 高并发资金处理

```java
// 方案：
1. Seata AT模式保证分布式事务
2. Redis分布式锁防并发
3. 乐观锁 + 状态机保证资金安全
4. 消息队列异步解耦
```

### 2. 跨系统数据同步

```java
// 方案：
1. 统一消息总线（RabbitMQ）
2. 消息幂等 + 补偿机制
3. Canal监听Binlog同步数据到ES
```

### 3. 实时风控

```java
// 方案：
1. 规则引擎（Drools）
2. 特征数据实时计算（Flink）
3. 决策树模型（机器学习）
```

## 项目成果

- 系统上线0故障，运行1年+
- 资金处理0风险，0资损
- 查询响应时间从15秒降到500毫秒
- 支持业务量增长10倍
