# Q5: 项目中JVM调优问题？如何排查和解决？

## 常见JVM问题

- **OOM（内存溢出）**：堆内存不足、内存泄漏、MetaSpace不足
- **频繁GC**：年轻代太小、大对象直接进老年代、内存碎片
- **GC停顿长**：老年代碎片化、FULL GC频繁
- **内存泄漏**：长连接未关闭、静态集合引用、ThreadLocal未清理

## 排查工具

- **jstat -gcutil PID 1000**：查看GC统计，每秒更新
- **jmap -heap PID**：查看堆配置和使用
- **jmap -dump:format=b,file=heap.hprof PID**：导出堆转储
- **MAT**：分析堆转储文件，定位内存泄漏
- **Arthas**：在线诊断，dashboard、heapdump、watch

## 实际案例：支付记录查询优化

### 问题背景
嘉立创EFS应收账款结算中台，600万+支付记录查询响应时间15秒+

### 排查过程
1. 通过EXPLAIN分析SQL执行计划，发现全表扫描
2. 定位到分区表在跨片排序时执行计划不优
3. 确认索引选择性低，部分查询走错索引
4. jstat发现年轻代频繁Minor GC

### 解决方案
1. **全局路由索引**：针对常用查询条件创建复合索引
2. **局部时间索引**：按时间范围快速定位数据
3. **索引下推**：在索引层过滤数据，减少回表次数
4. **Parallel Query**：开启MySQL并行查询

### 优化结果
查询响应时间：15秒 → 3秒，性能提升5倍
