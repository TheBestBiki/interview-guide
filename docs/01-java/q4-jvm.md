# Q4: JVM内存模型和垃圾回收算法？

## JVM内存模型（JMM）

| 区域 | 线程私有/共享 | 存储内容 | 异常 |
|------|--------------|----------|------|
| 堆（Heap） | 共享 | 对象实例、数组 | OutOfMemoryError |
| 栈（Stack） | 私有 | 局部变量、操作数栈 | StackOverflowError |
| 方法区 | 共享 | 类信息、常量、静态变量 | OutOfMemoryError |
| 本地方法栈 | 私有 | Native方法服务 | StackOverflowError |
| 程序计数器 | 私有 | 字节码行号指示器 | 无 |

## 堆内存划分

```
堆内存 = 新生代 + 老年代 + 元空间(1.8+)

新生代（1/3堆）：
  - Eden区（8/10）
  - Survivor区 From（1/10）
  - Survivor区 To（1/10）

老年代（2/3堆）
```

## 垃圾回收算法

### ① 标记-清除算法（Mark-Sweep）
- **步骤**：标记存活对象 → 清除死亡对象
- **缺点**：产生内存碎片

### ② 标记-整理算法（Mark-Compact）
- **步骤**：标记存活对象 → 移动到一端 → 清除边界外对象
- **优点**：无内存碎片
- **适用**：老年代

### ③ 复制算法（Copying）
- **步骤**：分成两块相等空间FROM/TO，FROM满时将存活对象复制到TO
- **优点**：无内存碎片，分配简单
- **缺点**：空间浪费50%
- **适用**：新生代（对象存活率低）

### ④ 分代收集算法
- **新生代**：Minor GC，使用复制算法
- **老年代**：Full/Major GC，使用标记-清除或标记-整理

## 垃圾回收器

| 回收器 | 特点 | 适用场景 |
|--------|------|----------|
| Serial | 单线程，Stop The World | client模式 |
| Parallel | 多线程，关注吞吐量 | 后台计算 |
| CMS | 并发标记清除，关注停顿时间 | web服务 |
| G1 | 区域化分代，平衡吞吐量和停顿 | 服务端 |
| ZGC | 超低延迟（<10ms） | 大内存低延迟 |

## 常见JVM参数

```bash
-Xms4g -Xmx4g                    # 堆初始和最大
-Xmn2g                              # 新生代大小
-XX:NewRatio=2                      # 老年代/新生代=2
-XX:+UseG1GC                        # 使用G1
-XX:MaxGCPauseMillis=200           # 最大GC停顿时间
```
