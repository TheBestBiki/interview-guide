# Q8: 如何保证微服务高可用？限流/熔断方案？

## 高可用策略

| 策略 | 作用 |
|------|------|
| 服务冗余 | 多实例部署，负载均衡 |
| 服务降级 | 核心功能优先，非核心功能熔断 |
| 限流 | 控制QPS，防止系统过载 |
| 熔断 | 快速失败，防止故障蔓延 |
| 隔离 | 线程池隔离、信号量隔离 |
| 超时重试 | 设置合理的超时和重试 |
| 幂等 | 接口幂等设计，防止重复请求 |
| 多级缓存 | 本地缓存+Redis，减少后端压力 |

## 限流算法

### ① 计数器算法
- 维护一个计数器，每请求+1
- 优点：简单
- 缺点：无法应对突发流量

### ② 滑动窗口
- 将时间窗口分成多个小窗口
- 优点：比计数器平滑

### ③ 漏桶算法
- 以固定速率处理请求
- 优点：请求均匀
- 缺点：突发请求需要等待

### ④ 令牌桶算法
- 以固定速率生成令牌，请求获取令牌
- 优点：允许突发流量

## Sentinel熔断策略

| 策略 | 触发条件 | 效果 |
|------|----------|------|
| 慢调用比例 | 响应时间 > 阈值，且比例 > 比例阈值 | 熔断 |
| 异常比例 | 异常比例 > 阈值 | 熔断 |
| 异常数 | 异常数量 > 阈值 | 熔断 |
